# Attestor v0.1.1-evaluation Release Notes

**Release type:** GitHub pre-release / CI-backed evaluation baseline

**Tag:** `v0.1.1-evaluation`

**Package version:** `0.1.1-evaluation`

**Primary reviewer path:** [Attestor Evaluation Packet v0.1](v0.1-evaluation-packet.md)

This release keeps the v0.1 evaluation scope intact and adds a GitHub-visible evaluation gate. It is meant to let an outside engineer see the same reviewer path run in CI before deeper architecture or production-readiness review.

## What Changed Since v0.1.0-evaluation

- Added the `Evaluation Smoke` GitHub Actions workflow for push and pull-request checks.
- Added the `Full Verify` GitHub Actions workflow for manual and weekly full verification.
- The smoke gate runs `npm ci`, TypeScript checking, evaluation packet docs tests, evaluation release notes tests, package-runner contract tests, the admission demo, the customer gate demo, and proof-surface rendering.
- The smoke gate uploads `.attestor/proof-surface/latest/` as the `proof-surface` workflow artifact.
- GitHub Actions are pinned to Node 24-capable major versions: `actions/checkout@v5`, `actions/setup-node@v5`, and `actions/upload-artifact@v6`.

## What To Run Locally

Start from a clean checkout at the release tag:

```bash
npm ci
npm run example:admission
npm run example:customer-gate
npm run proof:surface
```

For the portable proof-showcase path without external API credentials:

```bash
npm run showcase:proof
npm run verify:cert -- .attestor/showcase/latest/evidence/kit.json
```

`verify:cert` reports `PROOF_DEGRADED` and exits non-zero when the proof is local PostgreSQL-backed but upstream-model fixture-backed. That is expected for this evaluation release; use `npm run verify` as the green local release gate.

Before treating the release tag as reviewable:

```bash
npm run verify
```

## CI Evidence

The release branch now has two workflow gates:

- `Evaluation Smoke`: fast push/PR gate for the reviewer path and proof-surface artifact.
- `Full Verify`: manual and weekly scheduled gate for `npm run verify`.

The GitHub smoke gate is a trust signal for the evaluation packet. It does not replace production deployment rehearsal, security audit, or customer-operated validation.

## What This Evaluation Release Demonstrates

- A concrete finance wedge: AI-assisted output -> structured financial record release.
- A bounded decision vocabulary: `admit`, `narrow`, `review`, `block`.
- A short local admission path with one admitted consequence and one blocked fail-closed consequence.
- A customer-side gate that proceeds only when Attestor returns an allowed decision.
- Local proof-surface rendering under `.attestor/proof-surface/latest/`.
- A CI-backed proof-surface artifact from the `Evaluation Smoke` workflow.
- Independent kit inspection through `npm run verify:cert -- .attestor/showcase/latest/evidence/kit.json`, with expected `PROOF_DEGRADED` posture unless a live upstream model is configured.
- A full local verification gate through `npm run verify`.
